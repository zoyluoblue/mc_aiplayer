# AI 玩家 Fabric

面向 Minecraft Fabric 的 AI 玩家模组。

目标版本：

- Minecraft: `1.21.3`
- Fabric Loader: `0.18.4`
- Fabric API: `0.114.1+1.21.3`
- Java: `21`

## 构建

```bash
./gradlew build
```

构建产物：

```text
build/libs/aiplayer-fabric-1.0.0.jar
```

## 配置

配置文件统一使用：

```text
config/ai.json
```

开发环境执行 `runClient` 时，会自动把项目根目录下的 `config/ai.json` 复制到 `run/config/ai.json`。

可以参考 [config/ai.json.example](config/ai.json.example) 创建配置文件。

```json
{
  "deepseek": {
    "apiKey": "sk-...",
    "baseUrl": "https://api.deepseek.com",
    "model": "deepseek-v4-flash",
    "thinkingEnabled": false,
    "reasoningEffort": "high",
    "maxTokens": 8000,
    "temperature": 0.7
  },
  "behavior": {
    "actionTickDelay": 20,
    "enableChatResponses": true,
    "maxActiveAiPlayers": 10
  }
}
```

## 日志

AI 玩家会把模组内所有级别的运行日志按类别写入游戏目录下的 `log/` 目录，包括 `TRACE`、`DEBUG`、`INFO`、`WARN` 和 `ERROR`。开发环境执行 `runClient` 时，目录位置是：

```text
run/log/
```

每个类别使用一个子目录，例如 `action`、`planning`、`recipe`、`make_item`、`snapshot`、`player`、`llm`、`plugin`、`system` 等。每个类别最多保留 3 个 `.log` 文件，当前文件名为 `<category>.log`，历史文件为 `<category>-1.log` 和 `<category>-2.log`。单个文件最大 100MB；达到上限后自动 rotate，最旧文件会被删除。文件内容会包含时间、线程名、级别、类别、logger 名称、消息和异常堆栈。

每次通过 `/ai say ...` 发起自然语言任务时，日志会生成统一的 `taskId`，并贯穿 `planning`、`snapshot`、`recipe`、`action` 和 `make_item`。排查制作铁工具失败时，优先查看：

```text
run/log/planning/planning.log
run/log/recipe/recipe.log
run/log/action/action.log
run/log/make_item/make_item.log
run/log/snapshot/snapshot.log
```

先在 `planning.log` 找到本次命令的 `taskId`，再用同一个 `taskId` 到其他分类日志中追踪。`make_item.log` 会记录每个 step 的开始和结束，包括目标物品数量、AI 坐标、维度、主手工具、背包摘要、执行状态、失败原因以及是否允许重规划。

## 命令

```mcfunction
/ai spawn
```

为自己召唤一个默认名称的 AI 玩家。每名玩家最多只能拥有一个 AI 玩家。

```mcfunction
/ai spawn 小助手
```

为自己召唤一个名为“小助手”的 AI 玩家。

```mcfunction
/ai say 帮我在附近建一座小木屋
```

让你的 AI 玩家规划任务并执行。任务内容建议使用中文。

AI 玩家现在按生存玩家逻辑执行任务：初始没有材料，拥有 36 格物品背包，建造前会先砍树、把原木合成木板，再走到目标位置逐块放置并消耗背包材料；挖矿需要先收集木头制作工具，石头和矿物不会再凭空获得或瞬间完成。

```mcfunction
/ai say 去砍一棵树
```

只砍附近的一棵树，获得的原木会保留在 AI 背包里，不会自动合成为木板。

```mcfunction
/ai say 做一个门
```

递归制作木门：如果背包里没有足够木板，AI 会先观察背包和附近箱子，按已有材料或附近树木自动选择木门木种，解析配方链，再按 step 砍原木、合成木板、准备工作台材料并合成木门。每完成一个 step 后都会重新观察；如果材料变化、箱子新增材料或路径失败，会重新计算下一步。

```mcfunction
/ai say 帮我做一个木斧头
```

制作木制工具现在统一走 `make_item` 递归链，不再直接执行低层 `craft`。如果材料不足，AI 会查询木斧配方，自动补齐工作台、木板和木棍：缺原木时先砍树，缺木板时合成木板，缺木棍时合成木棍，最后再合成木斧。DeepSeek 即使返回旧的 `craft` 动作，本地也会自动重写成 `make_item`。

```mcfunction
/ai say 帮我做一把铁铲
```

铁制工具会递归追溯到熔炼和矿物采集。以铁铲为例，AI 会先补齐工作台、木镐、圆石、石镐、熔炉、煤和生铁，再用熔炉把生铁烧成铁锭，最后用铁锭和木棍合成铁铲。附近没有铁矿或煤矿时会给出明确失败原因。

```mcfunction
/ai say 帮我做个铁镐
```

铁镐会走同样的普通生存链：先做工作台和木镐，再优先采集 3 个圆石制作石镐，然后采集铁矿和煤矿，最后补齐熔炉、烧出铁锭并合成铁镐。配方解析会优先使用铁矿、煤矿这类自然基础来源，不会把 `minecraft:raw_iron_block` 或 `minecraft:coal_block` 当作默认自然资源。圆石采集找不到露天石头时，会有限地向下阶梯式挖开覆盖层；如果向下搜索后仍找不到可采集石头，会停止任务并提示移动到矿洞、低处或给 AI 放入圆石。矿石采集会优先选择露出空气面且可直接工作或可寻路接近的矿；如果追踪同一个矿点一段时间仍没有获得原矿，AI 会跳过该目标并切换到其他矿点或进入探矿，避免为了山体内部矿一路爬到高处。

木材类材料会按原版通用材料处理。`minecraft:oak_log` 和 `minecraft:oak_planks` 在工具、木棍、工作台等通用配方里代表“任意原木 / 任意木板”；AI 砍到云杉、白桦、丛林、金合欢、深色橡木、红树或樱花原木后，也会继续推进同一条制作链。树木目标不可达时，AI 会跳过该目标并寻找下一棵树，避免反复卡在同一处。

```mcfunction
/ai say 给我一把铁铲
```

“给我 / 我要 / 拿 / 来一把 / 弄 / 搞”这类明确索要物品的说法，也会优先走本地 `make_item` 生存链，不会先交给 DeepSeek 猜任务拆解。AI 执行采集、挖矿、放置方块和打水时会面向目标方块，手持合适工具并按节奏挥手，视觉上更接近普通玩家操作。

```mcfunction
/ai say 帮我烧一块牛肉
```

烧制食物会走生存链。以熟牛肉为例，AI 会先确认是否有熔炉、煤和生牛肉；缺熔炉时递归制作熔炉，缺煤时寻找煤矿，缺生牛肉时寻找附近的牛获取掉落物，最后再用熔炉烧成熟牛肉。

```mcfunction
/ai say 帮我打一桶水
```

打水会先制作空桶，再寻找水源方块。AI 会递归补齐铁桶所需的铁锭、熔炉、煤、生铁、石镐等前置材料，拿到空桶后走到附近水源并执行 `fill_water` step，把空桶变成水桶。

```mcfunction
/ai say 帮我做一把钻石镐
```

常规生存进阶链已扩展到木、石、铁、金、铜、钻石资源。以钻石镐为例，AI 会先完成木镐、石镐、铁矿和煤矿采集、熔炉、铁锭和铁镐，再去挖钻石矿，最后用钻石和木棍合成钻石镐。金矿、钻石、红石和绿宝石会要求先拥有铁镐；铁矿、铜矿和青金石会要求至少石镐；煤和圆石会要求至少木镐。

```mcfunction
/ai say 先砍树，再建一间小木屋
```

从零开始收集木材并建造小型生存木屋。

```mcfunction
/ai say 收集木头，然后挖一些石头
```

先收集木头制作基础工具，再像普通生存玩家一样挖石头。AI 会优先寻找露出空气、可接近的石头；如果附近石头被地形挡住或路径无法到达，会放弃该目标并换其他石头。附近没有可接近石头时，会从脚下向下挖开覆盖层，直到挖到石头获得圆石。

```mcfunction
/ai list
```

查看你当前拥有的 AI 玩家。

```mcfunction
/ai backpack
```

查看你的 AI 玩家当前背包物品，用来确认它已经收集到的材料和工具。

```mcfunction
/ai backpack take minecraft:oak_log 16
```

从 AI 背包取出指定物品到你的玩家背包。物品必须真实存在于 AI 背包中；如果你的背包空间不足，只会转移能放下的数量，剩余物品会回到 AI 背包。

```mcfunction
/ai backpack put minecraft:oak_log 16
```

把你玩家背包里的指定物品放入 AI 背包。物品必须真实存在于你的背包中；如果 AI 背包空间不足，只会放入能容纳的数量，剩余物品会回到你的背包。

```mcfunction
/ai snapshot
```

输出 AI 当前可观察状态 JSON，并同步写入日志。这个 JSON 包含玩家原始命令、AI 背包 36 格物品、手持物和装备、附近可访问箱子、关键方块、附近实体、当前位置、血量、维度、生物群系、昼夜和当前任务状态。后续 DeepSeek 规划会以这份结构化观察作为当前状态依据。

```mcfunction
/ai recipe minecraft:oak_door 1
```

查看目标物品的递归配方链。配方解析会先扣减 AI 背包，再识别附近可访问箱子中的材料，缺失部分继续追溯到基础资源。现在制作链不再依赖手写物品白名单：AI 会读取原版 `RecipeManager`，递归解析合成、熔炼、高炉、烟熏炉、营火烹饪、切石机和锻造台配方；同一物品存在多条配方或同一材料位接受多个物品时，会尝试不同原版配方变体，失败后回滚并继续尝试下一种。

基础资源由来源表负责，例如原木、可直接破坏获得的方块、圆石、煤、铜、铁、金、钻石、红石、青金石、绿宝石、黑曜石、远古残骸、下界石英、萤石粉、黏土球、燧石、线、甘蔗、竹子、仙人掌、海带、海草、藤蔓、蘑菇、南瓜、西瓜片和生牛肉。水桶这类没有合成配方的交互物品会被转换成生存 step，例如 `fill_water` 会要求先有空桶，再寻找水源装水。需要工作台、熔炉、高炉、烟熏炉、营火、切石机或锻造台的物品会把工作站作为非消耗需求处理；缺工作站时会继续递归制作工作站。

```mcfunction
/ai recipe minecraft:diamond_pickaxe 1
```

查看钻石镐从零开始的递归链，通常会展开为砍树、做木板、做木棍、做工作台、做木镐、先挖圆石做石镐、挖铁矿和煤、补圆石做熔炉、烧铁锭、做铁镐、挖钻石、合成钻石镐。

```mcfunction
/ai plan minecraft:oak_door 1
```

查看本地验证后的高层执行计划 JSON。计划只允许 `gather_tree`、`gather_stone`、`gather`、`craft_inventory`、`craft_station`、`fill_water`、`withdraw_chest`、`deposit_chest`、`place_block`、`return_to_owner` 等生存步骤；如果 DeepSeek 或调试输入给出错误物品 ID、创造模式能力或错误材料数量，本地验证层会拒绝或按原版配方链纠正。

```mcfunction
/ai stop
```

停止你的 AI 玩家当前任务并清空任务队列，随后它会按正常移动逻辑回到你身边。

```mcfunction
/ai recall
```

立即停止你的 AI 玩家当前任务和任务队列，并把它直接召回到你身边。这个命令不会等待寻路完成，适合 AI 卡住、距离太远或需要马上回到玩家身边时使用。

```mcfunction
/ai remove
```

移除你的 AI 玩家。

```mcfunction
/ai help
```

查看命令帮助。

## 客户端面板

进入游戏后按 `Alt+2` 打开 AI 面板，macOS 按 `Option+2`。输入 `spawn` 可以创建你的 AI 玩家，输入中文自然语言任务会转发给你的 AI 玩家。面板顶部的“背包”按钮会同时显示 AI 背包和玩家背包；“停止”按钮会让 AI 停止当前任务并按正常移动逻辑回到你身边；“召回”按钮会立即停止所有任务并把 AI 直接传送到你身边。输入 `snapshot` 可以输出当前观察 JSON，方便调试 AI 看到的背包、箱子和附近环境。

AI 面板背包支持双背包直接转移：先点击有物品的来源格，再点击另一个背包里的目标格。玩家背包物品可以放入 AI 背包指定格，AI 背包物品也可以取出到玩家背包指定格；左键目标格尽量转移整组，右键目标格只转移 1 个。所有转移都经过服务端校验，不会凭空生成物品。

客户端会自动关闭 Minecraft 的“失去焦点时暂停”选项。切换到其他窗口时，单人游戏不会因为窗口失焦而自动暂停；手动按 `Esc` 打开暂停菜单仍然会按原版逻辑暂停。

## 开发进度

- Phase 1 Observe：已完成。AI 会在每次规划前生成结构化观察 JSON，`/ai snapshot` 和 AI 面板输入 `snapshot` 可手动查看。
- Phase 2 RecipeResolver：已完成。`/ai recipe <item> [count]` 可以查看当前背包和附近箱子扣减后的递归材料链，`/ai say 做一个门` 会在日志中记录木门配方计划。
- Phase 3 DeepSeek Planner：已完成。新增高层计划 schema、计划解析、允许动作校验、非法物品拒绝和基于本地配方的材料数量纠正，`/ai plan <item> [count]` 可查看验证后的计划 JSON。
- Phase 4 Execute And Replan：已完成。新增 `make_item` 执行动作、step 执行器、目标检查和复盘策略；`/ai say 做一个门` 会按“观察 -> 配方 -> 高层 step -> 执行一个 step -> 重新观察”的循环执行，附近没有树时会给出明确失败原因。
- 制作路由修正：已完成。玩家说“做木斧头 / 做木镐 / 合成工作台 / 做门”等制作类任务时，本地会优先生成 `make_item`；DeepSeek 返回的旧 `craft` 动作也会被自动改写为 `make_item`，避免材料不足时直接失败。
- 配方扫描稳定性修正：已完成。已支持物品优先使用本地配方定义，RecipeManager 扫描遇到特殊配方异常时会跳过，避免制作任务触发服务端 tick 崩溃。
- AI 背包转移：已完成。支持 `/ai backpack take`、`/ai backpack put` 命令；AI 面板背包会同时显示 AI 背包和玩家背包，先点来源格再点目标格即可在两个背包之间转移物品。
- AI 召回：已完成。支持 `/ai recall` 和 AI 面板“召回”按钮，会立即清空当前任务并把 AI 直接移动到玩家身边。
- 原版配方递归：已完成。制作链升级为“原版 RecipeManager 配方递归 + 基础资源来源表”，不再用手写表白名单限制目标物品。已支持合成、熔炼、高炉、烟熏炉、营火烹饪、切石机和锻造台配方，并支持通用 `block:minecraft:*` 基础采集 step。当前仍会拒绝创造专属物品、无原版配方且来源表没有定义的战利品，以及需要复杂实体交互、村民交易、结构探索或跨维度寻路才能稳定获得的物品。
- 石头采集可达性修正：已完成。`gather stone` 和 `make_item` 中的 `gather_stone` 会优先选择可直接工作或可寻路接近的石头；目标卡住后会换目标，找不到可接近石头时会进入持续下挖模式，不会被暴露但不可达的下方石头拉回寻路循环。向下搜索会按当前朝向挖横向阶梯：先清理前方站位空间，再挖前方下一格，移动到可站立下降点后继续挖；达到上限或长时间没有新增圆石时会停止并给出明确失败原因。
- 分类 rotate 日志：已完成。模组会在 `run/log/` 按 `action`、`planning`、`recipe`、`make_item`、`snapshot`、`player`、`llm` 等类别保存所有级别日志；每个类别最多 3 个文件，单个文件最大 100MB，超过后自动 rotate。
- 三个生存案例验证：已完成。新增自动化测试覆盖“帮我做一把铁铲”“帮我烧一块牛肉”“帮我打一桶水”，确认中文目标识别、递归配方链和 `fill_water` 执行 step 都能生成；熟牛肉已支持牛掉落物来源，水桶已支持先做空桶再到水源装水。
- 铁镐失败链路修正：已完成。配方递归现在会优先使用显式基础资源来源，`minecraft:raw_iron` 会走 `iron_ore`，`minecraft:coal` 会走 `coal_ore`；通用 `block:minecraft:*` 来源只在快照中确实可见且可达时使用，避免铁镐计划错误地采集铁块或煤炭块。
- 行为拟真优化：已完成。AI 工作时会面向具体目标方块，手持当前任务所需工具，并使用带冷却的挥手动作；制作任务的周期性复盘不会再打断 `gather_stone` 和矿物采集这类有内部状态的长耗时 step。
- 生存挖矿日志基线：已完成。每次自然语言任务会生成统一 `taskId`，制作任务的每个 step 都会在 `make_item.log` 中记录开始、结束、背包摘要、坐标、维度、主手工具、执行状态和重规划标记，方便定位铁工具链路失败点。
- 配方工具优先顺序：已完成。递归计划会先补齐采集工具，再补齐熔炉等工作站；铁制工具会先做石镐再处理铁矿和熔炉，钻石工具会先做铁镐再挖钻石。
- 连续资源会话：已完成。`make_item` 在重算下一步时会保留本次任务的采集上下文；连续补圆石会复用刚挖出的矿洞和下挖状态，矿石采集也会记录已失败目标并跳过重复不可达位置。
- 阶梯式下挖：已完成。AI 找不到可接近石头时会用横向阶梯方式向下挖，避免优先垂直挖脚下；挖开前方站位和下一格后会移动到安全下降点继续采集。
- 矿石探矿兜底：已完成。铁矿、煤矿等矿物采集在附近没有可见矿石时不会立刻失败，会进入本地探矿模式，沿阶梯/矿道继续挖掘并周期性重新扫描；超过搜索预算后才报告具体失败原因。
- 木材等价与树木换目标：已完成。制作链中的通用原木和通用木板会接受所有原版木材变体；AI 背包已有 `minecraft:spruce_log` 等其他原木时，不会继续强制寻找 `minecraft:oak_log`。采树目标卡住后会记录不可达目标并切换下一棵树。
- 矿石目标筛选：已完成。AI 只会把露出空气面且可直接工作或可寻路接近的矿当作追踪目标；矿点长时间没有产出会被加入跳过列表并重新找矿或进入探矿。
