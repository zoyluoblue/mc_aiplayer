# AI 玩家 Fabric

面向 Minecraft Fabric 的 AI 玩家模组。

目标版本：

- Minecraft: `1.21.3`
- Fabric Loader: `0.18.4`
- Fabric API: `0.114.1+1.21.3`
- Java: `21`

## 构建

```bash
./gradlew build
```

构建产物：

```text
build/libs/aiplayer-fabric-1.0.0.jar
```

## 配置

配置文件统一使用：

```text
config/ai.json
```

开发环境执行 `runClient` 时，会自动把项目根目录下的 `config/ai.json` 复制到 `run/config/ai.json`。

可以参考 [config/ai.json.example](config/ai.json.example) 创建配置文件。

```json
{
  "deepseek": {
    "apiKey": "sk-...",
    "baseUrl": "https://api.deepseek.com",
    "model": "deepseek-v4-flash",
    "thinkingEnabled": false,
    "reasoningEffort": "high",
    "maxTokens": 8000,
    "temperature": 0.7
  },
  "behavior": {
    "actionTickDelay": 20,
    "enableChatResponses": true,
    "maxActiveAiPlayers": 10
  }
}
```

## 日志

AI 玩家会把模组内所有级别的运行日志按类别写入游戏目录下的 `log/` 目录，包括 `TRACE`、`DEBUG`、`INFO`、`WARN` 和 `ERROR`。开发环境执行 `runClient` 时，目录位置是：

```text
run/log/
```

每个类别使用一个子目录，例如 `action`、`planning`、`recipe`、`make_item`、`snapshot`、`player`、`llm`、`plugin`、`system` 等。每个类别最多保留 3 个 `.log` 文件，当前文件名为 `<category>.log`，历史文件为 `<category>-1.log` 和 `<category>-2.log`。单个文件最大 100MB；达到上限后自动 rotate，最旧文件会被删除。文件内容会包含时间、线程名、级别、类别、logger 名称、消息和异常堆栈。

每次通过 `/ai say ...` 发起自然语言任务时，日志会生成统一的 `taskId`，并贯穿 `planning`、`snapshot`、`recipe`、`action` 和 `make_item`。排查制作铁工具失败时，优先查看：

```text
run/log/planning/planning.log
run/log/recipe/recipe.log
run/log/action/action.log
run/log/make_item/make_item.log
run/log/mining/mining.log
run/log/mining_strategy/mining_strategy.log
run/log/snapshot/snapshot.log
```

先在 `planning.log` 找到本次命令的 `taskId`，再用同一个 `taskId` 到其他分类日志中追踪。`make_item.log` 会记录每个 step 的开始和结束，包括目标物品数量、AI 坐标、维度、主手工具、背包摘要、执行状态、失败原因以及是否允许重规划。`mining.log` 会记录挖矿运行事实，包括起点、起始 Y、目标高度策略、扫描半径、候选矿物数量、可达数量、拒绝原因、下挖方向、已挖方块数和探矿超时摘要。

长时间挖矿任务还会写入 `mining_strategy.log`。AI 在 `gather_stone`、铁矿、金矿、钻石等长 step 中每 30 秒最多向 DeepSeek 上报一次当前目标、当前 step、材料状态、附近方块、近期失败和近期事件；DeepSeek 只返回策略建议，本地代码会校验后再决定继续当前 step、换目标、切换阶梯下挖或重新观察规划。

## 命令

```mcfunction
/ai spawn
```

为自己召唤一个默认名称的 AI 玩家。每名玩家最多只能拥有一个 AI 玩家。

```mcfunction
/ai spawn 小助手
```

为自己召唤一个名为“小助手”的 AI 玩家。

```mcfunction
/ai say 帮我在附近建一座小木屋
```

让你的 AI 玩家规划任务并执行。任务内容建议使用中文。

AI 玩家现在按生存玩家逻辑执行任务：初始没有材料，拥有 36 格物品背包，建造前会先砍树、把原木合成木板，再走到目标位置逐块放置并消耗背包材料；挖矿需要先收集木头制作工具，石头和矿物不会再凭空获得或瞬间完成。

```mcfunction
/ai say 去砍一棵树
```

只砍附近的一棵树，获得的原木会保留在 AI 背包里，不会自动合成为木板。

```mcfunction
/ai say 做一个门
```

递归制作木门：如果背包里没有足够木板，AI 会先观察背包和附近箱子，按已有材料或附近树木自动选择木门木种，解析配方链，再按 step 砍原木、合成木板、准备工作台材料并合成木门。每完成一个 step 后都会重新观察；如果材料变化、箱子新增材料或路径失败，会重新计算下一步。

```mcfunction
/ai say 帮我做一个木斧头
```

制作木制工具现在统一走 `make_item` 递归链，不再直接执行低层 `craft`。如果材料不足，AI 会查询木斧配方，自动补齐工作台、木板和木棍：缺原木时先砍树，缺木板时合成木板，缺木棍时合成木棍，最后再合成木斧。DeepSeek 即使返回旧的 `craft` 动作，本地也会自动重写成 `make_item`。

```mcfunction
/ai say 帮我做一把铁铲
```

铁制工具会递归追溯到熔炼和矿物采集。以铁铲为例，AI 会先补齐工作台、木镐、圆石、石镐、熔炉、煤和生铁，再用熔炉把生铁烧成铁锭，最后用铁锭和木棍合成铁铲。附近没有铁矿或煤矿时会给出明确失败原因。

```mcfunction
/ai say 帮我做个铁镐
```

铁镐会走同样的普通生存链：先做工作台和木镐，再优先采集 3 个圆石制作石镐，然后采集铁矿和煤矿，最后补齐熔炉、烧出铁锭并合成铁镐。配方解析会优先使用铁矿、煤矿这类自然基础来源，不会把 `minecraft:raw_iron_block` 或 `minecraft:coal_block` 当作默认自然资源。圆石采集找不到露天石头时，会有限地向下阶梯式挖开覆盖层；如果向下搜索后仍找不到可采集石头，会停止任务并提示移动到矿洞、低处或给 AI 放入圆石。矿石采集会优先选择露出空气面且可直接工作或可寻路接近的矿；如果追踪同一个矿点一段时间仍没有获得原矿，AI 会跳过该目标并切换到其他矿点或进入探矿，避免为了山体内部矿一路爬到高处。

木材类材料会按原版通用材料处理。`minecraft:oak_log` 和 `minecraft:oak_planks` 在工具、木棍、工作台等通用配方里代表“任意原木 / 任意木板”；AI 砍到云杉、白桦、丛林、金合欢、深色橡木、红树或樱花原木后，也会继续推进同一条制作链。树木目标不可达时，AI 会跳过该目标并寻找下一棵树，避免反复卡在同一处。

```mcfunction
/ai say 给我一把铁铲
```

“给我 / 我要 / 拿 / 来一把 / 弄 / 搞 / 挖 / 找 / 采集”这类明确索要物品或矿物的说法，也会优先走本地 `make_item` 生存链，不会先交给 DeepSeek 猜任务拆解。AI 执行采集、挖矿、放置方块和打水时会面向目标方块，手持合适工具并按节奏挥手，视觉上更接近普通玩家操作。

```mcfunction
/ai mining start iron 3
```

启动自动挖矿入口，目标仍按普通生存链执行。上例会让 AI 尝试取得 `minecraft:raw_iron x3`；`diamond` 会先补铁镐再挖钻石，`gold` 会先补铁镐再挖金矿，`obsidian` 会先补钻石镐再寻找黑曜石。`gold_ingot`、`iron_ingot`、`copper_ingot` 会被当作“先挖粗矿再熔炼”的目标，不会寻找不存在的锭方块。`diamond_ore` 这类矿石方块本体暂不支持，因为需要精准采集工具链；普通挖矿请使用 `diamond`。常用目标包括 `coal`、`copper`、`iron`、`gold`、`diamond`、`redstone`、`lapis`、`emerald`、`obsidian`、`ancient_debris`。自然语言里的“自动挖矿 / 去挖矿 / 挖一些矿”会归一到默认铁矿采集链。

```mcfunction
/ai mining status
```

查看当前自动挖矿状态，包含 taskId、目标、当前动作、是否执行/规划、坐标、Y 高度、主手物品和 AI 背包摘要。AI 面板输入 `mining status` 也会触发这个查询。

```mcfunction
/ai status
```

查看 AI 通用状态，包含当前阶段、目标、动作、执行/规划状态、位置、Y 高度、主手物品和背包摘要。AI 面板顶部的“状态”按钮会触发这个命令。

```mcfunction
/ai location
```

查看 AI 当前维度、坐标和与你的距离。AI 面板顶部的“定位”按钮会触发这个命令。

```mcfunction
/ai mining stop
```

停止当前自动挖矿任务，已获得的材料仍保留在 AI 背包中。

```mcfunction
/ai mining return
```

停止挖矿并让 AI 按正常移动回到玩家身边，不会瞬移。

```mcfunction
/ai say 帮我烧一块牛肉
```

烧制食物会走生存链。以熟牛肉为例，AI 会先确认是否有熔炉、煤和生牛肉；缺熔炉时递归制作熔炉，缺煤时寻找煤矿，缺生牛肉时寻找附近的牛获取掉落物，最后再用熔炉烧成熟牛肉。

```mcfunction
/ai say 帮我打一桶水
```

打水会先制作空桶，再寻找水源方块。AI 会递归补齐铁桶所需的铁锭、熔炉、煤、生铁、石镐等前置材料，拿到空桶后走到附近水源并执行 `fill_water` step，把空桶变成水桶。

```mcfunction
/ai say 帮我做一把钻石镐
```

常规生存进阶链已扩展到木、石、铁、金、铜、钻石资源。以钻石镐为例，AI 会先完成木镐、石镐、铁矿和煤矿采集、熔炉、铁锭和铁镐，再去挖钻石矿，最后用钻石和木棍合成钻石镐。金矿、钻石、红石和绿宝石会要求先拥有铁镐；铁矿、铜矿和青金石会要求至少石镐；煤和圆石会要求至少木镐。

```mcfunction
/ai say 帮我挖两块金锭
```

“挖金锭”会被理解为取得 `minecraft:gold_ingot`，而不是直接挖一个不存在的金锭方块。AI 会按生存链先确保铁镐，再挖金矿获得粗金，补煤和熔炉后烧成金锭。DeepSeek 如果返回旧的 `mine gold` 任务，本地也会自动重写为对应的 `make_item` 生存链。金矿目标会使用矿物资料表中的主世界维度和推荐 Y 范围；如果 AI 还在高海拔地表，会优先阶梯式下探，而不是长时间在地表随机扫描。

金锭这类长挖矿任务执行时，AI 会在挖石头、找铁矿、找金矿等阶段周期性把当前背包、附近箱子、当前位置、目标任务和最近失败上报给 DeepSeek，请它给出“继续、换目标、阶梯下挖、重新观察或请求玩家协助”这类策略建议。建议不会直接生成物品或移动坐标，仍由本地生存动作执行。

AI 会把已经成功完成的制作/采集链沉淀成技能记忆。比如一次成功的金锭任务会记录高层步骤、事件证据和曾经遇到的失败，后续同类任务会把这些经验摘要放进 DeepSeek 上下文，帮助它少重复错误路线。技能记忆只是经验提示，不会绕过本地配方、背包、箱子、工具和世界方块校验；最终是否完成仍由本地 Critic 和 `GoalChecker` 按真实背包判定。

```mcfunction
/ai say 帮我挖黑曜石
```

黑曜石、远古残骸、下界石英、下界金矿、煤、铜、铁、金、钻石、红石、青金石和绿宝石现在统一使用矿物资料表推演工具等级、目标方块、掉落物、维度和推荐高度。黑曜石和远古残骸会先递归制作钻石镐；下界石英和下界金矿要求在下界获取；远古残骸会在下界低高度探矿；黑曜石使用特殊环境规则，不会被普通矿道逻辑凭空推断出来。如果附近确实没有可见且可接近的特殊矿物，AI 会明确报告缺少目标来源。

```mcfunction
/ai say 先砍树，再建一间小木屋
```

从零开始收集木材并建造小型生存木屋。

```mcfunction
/ai say 收集木头，然后挖一些石头
```

先收集木头制作基础工具，再像普通生存玩家一样挖石头。AI 会优先寻找露出空气、可接近的石头；如果附近石头被地形挡住或路径无法到达，会放弃该目标并换其他石头。附近没有可接近石头时，会从脚下向下挖开覆盖层，直到挖到石头获得圆石。

```mcfunction
/ai list
```

查看你当前拥有的 AI 玩家。

```mcfunction
/ai backpack
```

查看你的 AI 玩家当前背包物品，用来确认它已经收集到的材料和工具。

```mcfunction
/ai backpack take minecraft:oak_log 16
```

从 AI 背包取出指定物品到你的玩家背包。物品必须真实存在于 AI 背包中；如果你的背包空间不足，只会转移能放下的数量，剩余物品会回到 AI 背包。

```mcfunction
/ai backpack put minecraft:oak_log 16
```

把你玩家背包里的指定物品放入 AI 背包。物品必须真实存在于你的背包中；如果 AI 背包空间不足，只会放入能容纳的数量，剩余物品会回到你的背包。

```mcfunction
/ai snapshot
```

输出 AI 当前可观察状态 JSON，并同步写入日志。这个 JSON 包含玩家原始命令、AI 背包 36 格物品、手持物和装备、附近可访问箱子、关键方块、附近实体、当前位置、血量、维度、生物群系、昼夜和当前任务状态。后续 DeepSeek 规划会以这份结构化观察作为当前状态依据。

```mcfunction
/ai recipe minecraft:oak_door 1
```

查看目标物品的递归配方链。配方解析会先扣减 AI 背包，再识别附近可访问箱子中的材料，缺失部分继续追溯到基础资源。现在制作链不再依赖手写物品白名单：AI 会读取原版 `RecipeManager`，递归解析合成、熔炼、高炉、烟熏炉、营火烹饪、切石机和锻造台配方；同一物品存在多条配方或同一材料位接受多个物品时，会尝试不同原版配方变体，失败后回滚并继续尝试下一种。

基础资源由来源表负责，例如原木、可直接破坏获得的方块、圆石、煤、铜、铁、金、钻石、红石、青金石、绿宝石、黑曜石、远古残骸、下界石英、萤石粉、黏土球、燧石、线、甘蔗、竹子、仙人掌、海带、海草、藤蔓、蘑菇、南瓜、西瓜片和生牛肉。水桶这类没有合成配方的交互物品会被转换成生存 step，例如 `fill_water` 会要求先有空桶，再寻找水源装水。需要工作台、熔炉、高炉、烟熏炉、营火、切石机或锻造台的物品会把工作站作为非消耗需求处理；缺工作站时会继续递归制作工作站。

```mcfunction
/ai recipe minecraft:diamond_pickaxe 1
```

查看钻石镐从零开始的递归链，通常会展开为砍树、做木板、做木棍、做工作台、做木镐、先挖圆石做石镐、挖铁矿和煤、补圆石做熔炉、烧铁锭、做铁镐、挖钻石、合成钻石镐。

```mcfunction
/ai plan minecraft:oak_door 1
```

查看本地验证后的高层执行计划 JSON。计划只允许 `gather_tree`、`gather_stone`、`gather`、`craft_inventory`、`craft_station`、`fill_water`、`withdraw_chest`、`deposit_chest`、`place_block`、`return_to_owner` 等生存步骤；如果 DeepSeek 或调试输入给出错误物品 ID、创造模式能力或错误材料数量，本地验证层会拒绝或按原版配方链纠正。

```mcfunction
/ai stop
```

停止你的 AI 玩家当前任务并清空任务队列，随后它会按正常移动逻辑回到你身边。

```mcfunction
/ai recall
```

立即停止你的 AI 玩家当前任务和任务队列，并把它直接召回到你身边。这个命令不会等待寻路完成，适合 AI 卡住、距离太远或需要马上回到玩家身边时使用。

```mcfunction
/ai remove
```

移除你的 AI 玩家。

```mcfunction
/ai help
```

查看命令帮助。

## 客户端面板

进入游戏后按 `Alt+2` 打开 AI 面板，macOS 按 `Option+2`。输入 `spawn` 可以创建你的 AI 玩家，输入中文自然语言任务会转发给你的 AI 玩家。面板顶部的“背包”按钮会同时显示 AI 背包和玩家背包；“状态”按钮会显示 AI 当前阶段、目标、动作、位置、主手和背包摘要；“定位”按钮会显示 AI 当前维度、坐标和与你的距离；“召回”按钮会立即停止所有任务并把 AI 直接传送到你身边。输入 `snapshot` 可以输出当前观察 JSON，方便调试 AI 看到的背包、箱子和附近环境。

AI 面板背包支持双背包直接转移：先点击有物品的来源格，再点击另一个背包里的目标格。玩家背包物品可以放入 AI 背包指定格，AI 背包物品也可以取出到玩家背包指定格；左键目标格尽量转移整组，右键目标格只转移 1 个。所有转移都经过服务端校验，不会凭空生成物品。

客户端会自动关闭 Minecraft 的“失去焦点时暂停”选项。切换到其他窗口时，单人游戏不会因为窗口失焦而自动暂停；手动按 `Esc` 打开暂停菜单仍然会按原版逻辑暂停。

## 开发进度

- Observe：已完成。AI 会在每次规划前生成结构化观察 JSON，`/ai snapshot` 和 AI 面板输入 `snapshot` 可手动查看。
- RecipeResolver：已完成。`/ai recipe <item> [count]` 可以查看当前背包和附近箱子扣减后的递归材料链，`/ai say 做一个门` 会在日志中记录木门配方计划。
- DeepSeek Planner：已完成。新增高层计划 schema、计划解析、允许动作校验、非法物品拒绝和基于本地配方的材料数量纠正，`/ai plan <item> [count]` 可查看验证后的计划 JSON。
- Execute And Replan：已完成。新增 `make_item` 执行动作、step 执行器、目标检查和复盘策略；`/ai say 做一个门` 会按“观察 -> 配方 -> 高层 step -> 执行一个 step -> 重新观察”的循环执行，附近没有树时会给出明确失败原因。
- 制作路由修正：已完成。玩家说“做木斧头 / 做木镐 / 合成工作台 / 做门”等制作类任务时，本地会优先生成 `make_item`；DeepSeek 返回的旧 `craft` 动作也会被自动改写为 `make_item`，避免材料不足时直接失败。
- 配方扫描稳定性修正：已完成。已支持物品优先使用本地配方定义，RecipeManager 扫描遇到特殊配方异常时会跳过，避免制作任务触发服务端 tick 崩溃。
- AI 背包转移：已完成。支持 `/ai backpack take`、`/ai backpack put` 命令；AI 面板背包会同时显示 AI 背包和玩家背包，先点来源格再点目标格即可在两个背包之间转移物品。
- AI 召回：已完成。支持 `/ai recall` 和 AI 面板“召回”按钮，会立即清空当前任务并把 AI 直接移动到玩家身边。
- 原版配方递归：已完成。制作链升级为“原版 RecipeManager 配方递归 + 基础资源来源表”，不再用手写表白名单限制目标物品。已支持合成、熔炼、高炉、烟熏炉、营火烹饪、切石机和锻造台配方，并支持通用 `block:minecraft:*` 基础采集 step。当前仍会拒绝创造专属物品、无原版配方且来源表没有定义的战利品，以及需要复杂实体交互、村民交易、结构探索或跨维度寻路才能稳定获得的物品。
- 石头采集可达性修正：已完成。`gather stone` 和 `make_item` 中的 `gather_stone` 会优先选择可直接工作或可寻路接近的石头；目标卡住后会换目标，找不到可接近石头时会进入持续下挖模式，不会被暴露但不可达的下方石头拉回寻路循环。向下搜索会按当前朝向挖横向阶梯：先清理前方站位空间，再挖前方下一格，移动到可站立下降点后继续挖；如果下方可站立点 120 tick 内没有移动进展，会记录并拒绝该位置，改为就地继续挖阶梯；达到上限或长时间没有新增圆石时会停止并给出明确失败原因。
- 分类 rotate 日志：已完成。模组会在 `run/log/` 按 `action`、`planning`、`recipe`、`make_item`、`snapshot`、`player`、`llm` 等类别保存所有级别日志；每个类别最多 3 个文件，单个文件最大 100MB，超过后自动 rotate。
- 铁镐失败链路修正：已完成。配方递归现在会优先使用显式基础资源来源，`minecraft:raw_iron` 会走 `iron_ore`，`minecraft:coal` 会走 `coal_ore`；通用 `block:minecraft:*` 来源只在快照中确实可见且可达时使用，避免铁镐计划错误地采集铁块或煤炭块。
- 行为拟真优化：已完成。AI 工作时会面向具体目标方块，手持当前任务所需工具，并使用带冷却的挥手动作；制作任务的周期性复盘不会再打断 `gather_stone` 和矿物采集这类有内部状态的长耗时 step。
- 工具耐久与动作拟真：已完成。AI 会按方块类型选择镐、斧、铲或空手，破坏方块后扣除真实背包工具耐久，耐久耗尽会移除工具；挖掘速度会随工具等级变化。放置方块前会先把对应方块拿到手上，攻击生物时会面向目标并按冷却节奏挥动武器。
- 工作站生命周期：已完成。需要工作台、熔炉、高炉、烟熏炉、营火、切石机或锻造台时，AI 会先检查附近是否已有对应工作站；没有时会递归制作并真实放置到身边，放置会消耗 AI 背包里的工作站物品。烧炼类任务会使用更长的处理节奏，并在消耗原料和燃料后产出结果。
- DeepSeek 复盘与本地兜底：已完成。制作 step 失败时会生成结构化失败复盘，记录失败分类、建议策略和建议动作；DeepSeek 只能给策略级建议，动作必须在白名单中，非法建议会被拒绝并改用本地兜底策略。
- Agent 回归矩阵：已完成。新增 12 条固定自然语言/控制命令评测，覆盖铁工具、金锭、黑曜石、熟牛肉、水桶、木门、砍树、停止、召回和闲聊，用来验证 DeepSeek Agent 入口不会退化成只会聊天或只会硬编码。
- Agent 意图归一：已完成。自然语言会先被解析为统一 `AgentIntent`，区分制作、烧炼、打水、采集、停止、召回、背包和闲聊，常见中文说法会归一到同一个目标物品或控制动作。
- Agent 上下文：已完成。发送给 DeepSeek 或复盘器的上下文统一包含当前目标、step、AI 背包/附近箱子摘要、附近关键方块和实体、近期失败、近期事件与允许动作，不包含 API key。
- Agent 动作白名单：已完成。DeepSeek 只能从 `ActionManifest` 声明的生存动作中选择策略或 step，动作参数会被校验；`teleport`、`give`、`setblock` 等作弊式能力不会执行。
- Agent 计划协议 V2：已完成。高层计划可表达为 `AgentPlan` JSON，每个 step 都带有 `stepId`、动作、参数、期望结果、所需事实和失败策略，便于执行中断后继续复盘。
- Agent 计划守卫：已完成。DeepSeek 计划会被转换为结构化校验结果；非法动作、作弊能力、非法物品和错误数量会产生 `PlanViolation`，并给出修复建议或回退到本地配方计划。
- Agent step 事件：已完成。制作任务的每个 step 会生成 `StepExecutionEvent`，记录状态、进度、tick 时间、背包变化、位置和消息，日志可直接用于 DeepSeek 复盘和玩家排查。
- Agent loop 调度器：已完成。新增统一观察/复盘判定逻辑，step 完成、背包或箱子变化、卡住、危险、时间片结束和连续失败都会触发重新观察或复盘。
- Agent 失败恢复：已完成。失败会被分类为材料、工具、配方、路径、资源、危险、背包、工作站或非法 DeepSeek 输出，并映射到换目标、补工具、继续探矿、放置工作站、回玩家身边或求助等策略。
- Agent 会话记忆：已完成。`AgentSession` 会保存当前任务、最近命令、近期失败、已拒绝目标、已知箱子/工作站和玩家偏好，并能生成 DeepSeek 上下文摘要。
- Agent 技能记忆：已完成。AI 会把成功的 `make_item` 任务保存为技能记录，包含目标、成功 step、执行事件和失败规避信息；后续 DeepSeek 规划、失败复盘和挖矿策略心跳会收到相关技能摘要，但当前世界事实和动作执行仍由本地代码校验。
- 本地 Critic：已完成。`make_item` 成功前会用本地 `GoalChecker` 检查真实背包数量；计划结束但目标未完成时，会返回 Critic 的事实说明，而不是让 DeepSeek 判定完成。
- Agent 评测基准：已完成。固定回归基准覆盖自然语言意图、控制命令、计划守卫、配方链和旧 TaskPlanner 入口，后续改动需要用该矩阵防止 Agent 能力退化。
- 生存挖矿日志基线：已完成。每次自然语言任务会生成统一 `taskId`，制作任务的每个 step 都会在 `make_item.log` 中记录开始、结束、背包摘要、坐标、维度、主手工具、执行状态和重规划标记，方便定位铁工具链路失败点。
- 挖矿回放日志：已完成。矿物采集和 `gather_stone` 会建立 `MiningRun`，执行事实单独写入 `run/log/mining/mining.log`；探矿失败会附带访问过的 Y 层、扫描次数、候选/可达/拒绝统计、拒绝原因、已挖方块数、破坏失败次数和疑似洞穴数量，便于判断失败是高度、路线、扫描、路径还是工具问题。
- 可复用矿井入口：已完成。挖石头和矿物时会记录共享矿井 waypoint、最近站位和洞穴入口；同一制作任务里的后续矿物 step 会尝试复用已有矿井位置。阶梯下探会跳过水、岩浆、基岩、砂砾、沙子和红沙，减少垂直坑、坍塌和危险方块导致的卡死。
- 分支矿道搜索：已完成。对标记为 `branchMinePreferred` 的矿物，AI 会先下探到路线计划的目标高度附近，再开 2 格高水平分支矿道；每段达到上限或受阻后换方向继续搜索。分支矿道继续写入 `mining.log`，并复用 waypoint、洞穴入口和矿物扫描统计。
- 洞穴优先与矿脉追踪：已完成。观察快照会记录附近可达洞穴入口、连通空气、暴露石壁和可见矿物数量；探矿时会优先进入已发现的安全洞穴入口扫描。挖到矿后，AI 会优先追踪相邻同类矿石，尽量连续采完整条暴露矿脉。
- 配方工具优先顺序：已完成。递归计划会先补齐采集工具，再补齐熔炉等工作站；铁制工具会先做石镐再处理铁矿和熔炉，钻石工具会先做铁镐再挖钻石。
- 连续资源会话：已完成。`make_item` 在重算下一步时会保留本次任务的采集上下文；连续补圆石会复用刚挖出的矿洞和下挖状态，矿石采集也会记录已失败目标并跳过重复不可达位置。
- 阶梯式下挖：已完成。AI 找不到可接近石头时会用横向阶梯方式向下挖，避免优先垂直挖脚下；下探会先选择可破坏的阶梯方块，而不是先追逐可能不可达的低处站位；挖开前方站位和下一格后会移动到安全下降点继续采集。
- 统一矿物资料表：已完成。煤、铜、铁、金、钻石、红石、青金石、绿宝石、下界石英、下界金矿、黑曜石和远古残骸统一记录目标方块、产物、工具等级和是否允许探矿；自然语言“挖/找/采集”会优先进入本地生存链，DeepSeek 返回的旧 `mine` 矿物任务也会被重写为 `make_item`。
- 矿石探矿兜底：已完成。铁矿、煤矿、金矿、钻石矿、红石矿、青金石矿和绿宝石矿在附近没有可见矿石时不会立刻失败，会进入本地探矿模式，沿阶梯/矿道继续挖掘并周期性重新扫描；超过搜索预算后才报告具体失败原因。
- 木材等价与树木换目标：已完成。制作链中的通用原木和通用木板会接受所有原版木材变体；AI 背包已有 `minecraft:spruce_log` 等其他原木时，不会继续强制寻找 `minecraft:oak_log`。采树目标卡住后会记录不可达目标并切换下一棵树。
- 矿石目标筛选：已完成。AI 只会把露出空气面且可直接工作或可寻路接近的矿当作追踪目标；矿点长时间没有产出会被加入跳过列表并重新找矿或进入探矿。
- DeepSeek 挖矿策略循环：已完成首批能力。所有 DeepSeek 请求共享 Minecraft 生存上下文；长挖矿 step 每 30 秒最多异步上报一次材料状态和目标任务；返回策略必须通过本地仲裁，只能触发继续、换目标、阶梯下挖或本地重建路线，不能绕过生存规则。
- 矿物高度和路线策略：已完成。矿物资料表现在记录维度、推荐 Y 范围、路线提示、特殊环境和探矿预算；金、钻石、红石、远古残骸等深层目标会先进入合理高度范围，再扫描或探矿。
- 矿物 Profile V2：已完成。每种矿物现在有 `primaryRange`、`fallbackRange`、`surfaceAllowed`、`cavePreferred` 和 `branchMinePreferred`。铁矿能表达地下常规铁和山地高处铁，金矿区分普通地下金和恶地高处金，绿宝石优先山地/高海拔，钻石和红石优先深层分支矿道；这些资料会发送给 DeepSeek 做策略参考，但最终执行仍由本地生存规则决定。
- 自动挖矿入口：已完成。新增 `/ai mining start <target> [count]`，支持煤、铜、铁、金、钻石、红石、青金石、绿宝石、黑曜石和远古残骸等目标别名；命令入口会统一转成 `make_item` 生存链，目标不会凭空生成。
- 矿工状态日志：已完成。自动挖矿会在 `mining.log` 记录准备工具、前往矿区、下探、洞穴扫描、分支矿道、采矿、等待玩家和完成等状态切换，并附带切换原因、位置、Y 高度和当前模式集合。
- 挖矿补给检查：已完成。自动挖矿会在开挖前检查最低镐等级、镐耐久和背包空间；深层分支矿道目标会尝试准备工作台、熔炉和燃料状态，并把结果写入 `mining.log`。背包满或镐耐久过低时会停止并等待玩家处理，避免挖矿产物丢失或工具中途损坏。
- 矿洞路线计划：已完成。每个挖矿 run 会生成入口、目标 Y 层、主/备用高度段、主方向、分支段长和最大转向次数，并在分支矿道推进或换向时记录每条分支的方向、Y 高度、长度和结果。
- 矿层下探修正：已完成。分支矿道不再因为铁矿 primary 范围过宽而停留在高海拔；铁矿等 `branchMinePreferred` 目标会先下探到路线目标 Y 附近。扫描到没有空气面的埋藏矿时会记录 embedded ore hint，并可用作分支方向参考；分支四向受阻时会换到更低矿层继续，而不是立刻失败。
- 洞穴优先入口：已完成。洞穴偏好的矿物会在挖矿开始时扫描附近可达洞口，跳过空气量不足、石壁暴露不足或附近敌对生物过多的入口，选择更有暴露面和可见矿物的洞穴作为优先扫描点。
- 挖矿危险处理：已完成。自动挖矿会节流检查低血量、附近敌对生物过多和导航卡住；低血量或怪物过多时会尝试返回玩家身边。挖掘目标周围发现岩浆、水、脚下垂直挖掘或坠落风险时，会记录危险并换目标或换分支。
- 挖矿状态命令：已完成。新增 `/ai status`、`/ai location`、`/ai mining status`、`/ai mining stop` 和 `/ai mining return`，AI 面板移除“停止”按钮并保留“状态”按钮，新增“定位”按钮。玩家可以查看 AI 当前阶段、坐标、自动挖矿当前目标与位置，停止挖矿任务，或让 AI 正常走回身边。
- 挖矿回归记录：已完成。新增 `docs/mining-regression.md`，记录金锭、钻石、红石、绿宝石、黑曜石和远古残骸的实机验收矩阵、日志检查规则和允许失败边界。
